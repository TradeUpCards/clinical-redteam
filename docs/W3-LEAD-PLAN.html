Branch policy (FLIPPED at end of Phase 1a)

Phase 1a (now complete): direct-to-main was allowed for solo Tate work. That allowance has been removed — permission rules in .claude/settings.local.json have been deleted.

Phase 1b onward: every code commit goes on a crt/<short-desc> feature branch, pushed with auto-MR flags:

git push -u origin crt/<branch> \
  -o merge_request.create \
  -o merge_request.target=main \
  -o merge_request.remove_source_branch=true

User merges the MR on GitLab, then git push github main mirrors to GitHub. Documentation / handoff regeneration remains the direct-to-main exception per Tate kickoff.

No worktrees by default per .gauntlet/week3/in-flight.md. For true wall-clock parallelism between Aria and Bram, EITHER dispatch them as background agents with isolation: "worktree", OR set up two worktrees manually.
